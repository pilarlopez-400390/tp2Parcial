Flujo end-to-end

Quien puede confirmar o realizar un turno? El tutor asignado puede confirmar o marcar realizado sus turnos; el administrador puede intervenir sobre cualquier turno.
Esto se hace con un middleware de autenticacion y autorizacion. Con AUTORIZAR se define quien puede hacer que accion, con AUTENTICAR se define quien es el usuario que esta haciendo la accion.
El middleware recibe un JWT, lo valida, y luego con la informacion del JWT se puede saber quien es el usuario y que rol tiene para autorizar o no la accion.
El JWT se genera al hacer login, y se guarda en el cliente (localStorage o cookies) para enviarlo en cada request que requiera autenticacion.

TIP: En uno de los servicios, hacer un archivo JSON que cosas puede hacer cada rol. Que acciones puede hacer quien. Tambien puede estar en una tabla en la base de datos, pero para este caso se pide un archivo JSON.
Los parametros van por query.
